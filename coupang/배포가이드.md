# 쿠팡 지정출고 — 웹 배포 (OV5와 같은 레포·Supabase 재사용)

이 폴더(`coupang/`)는 **OV5와 같은 GitHub 레포·Supabase 프로젝트를 그대로 재사용**합니다.
→ **새 레포·새 Supabase·새 SQL이 전혀 필요 없습니다.** Streamlit 앱만 하나 더 만들면 끝.

## 공유되는 것
- **대상품목(기능2)** : Supabase `app_settings` 테이블, key=`coupang_targets` (OV5의 `ov5` 키와 같은 테이블 공존)
- **하대 마스터** : `coupang/master_item.xlsx` (레포에 포함, 사이드바에서 Item_*.xlsx 업로드로 갱신 — 재배포 전까지 유지)
- 결과 파일 : 다운로드 버튼

## 배포 (Streamlit Cloud에서 앱 하나 더 추가)
1. https://share.streamlit.io → **Create app**
2. 설정:
   - Repository: `leehaesung999/ov5-shipment`  (OV5와 동일 레포)
   - Branch: `main`
   - **Main file path: `coupang/streamlit_app.py`**
3. **Advanced settings → Secrets** 에 OV5와 **똑같은** 2줄 붙여넣기:
   ```toml
   SUPABASE_URL = "https://txyrascjeczatqtgyuxn.supabase.co"
   SUPABASE_KEY = "eyJ...anon..."
   ```
4. **Deploy** → 사이드바에 **🟢 공유 모드 · Supabase (대상품목 공유)** 뜨면 성공

## 속도 개선 반영
- 하대 마스터(11k행) **mtime 캐시** → 매칭마다 재파싱 안 함
- 재고/주문 엑셀 **read_only + reset_dimensions** → 파싱 속도 대폭↑ (결과 동일 검증)
- 같은 파일 재클릭은 Streamlit 캐시로 즉시

## 로컬/LAN
비밀키 없이 실행하면 기존처럼 **로컬 파일 모드**(대상품목.json). `쿠팡 지정출고 매칭.bat` 그대로 사용.
