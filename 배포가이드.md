# OV5 지정출고 — 웹 배포 가이드 (무료, 여러 사용자 공유)

재고모니터(jaegomonitor)와 **같은 방식**입니다: GitHub → Streamlit Cloud → Supabase.
배포하면 `https://...streamlit.app` 주소로 **누구나 인터넷으로 접속**, 로트 지정·체크박스가
**모두에게 공유·영구 저장**됩니다.

> 비밀키(SUPABASE_URL/KEY)를 **안 넣으면** 그대로 **로컬 파일 모드**로 동작합니다.
> (지금 내 PC / 사내 LAN 실행은 아무것도 안 바꿔도 됨)

---

## 준비물 (모두 무료)
- GitHub: https://github.com
- Supabase: https://supabase.com
- Streamlit Cloud: https://share.streamlit.io

---

## STEP 1 — Supabase 데이터베이스 설정

1. https://supabase.com → **Start your project** → 로그인
2. **New project** → 이름(예: `ov5`) · DB 비밀번호 설정 → Create
3. 왼쪽 **SQL Editor** → 아래 SQL 실행:

```sql
-- 로트 지정 거래처 (수동 업로드 + 자동 누적/화면 체크 공유)
create table lot_assignments (
  id         bigserial primary key,
  item_code  text not null,
  expiry     text default '',          -- YYYYMMDD, 전체적용이면 ''
  category   text not null,            -- '농협' 또는 '농협,대리점' 처럼 콤마 다중
  item_name  text default '',
  source     text not null default 'auto',  -- 'manual'(업로드) | 'auto'(누적·체크)
  updated_at timestamptz default now(),
  unique (item_code, expiry, source)
);

-- 설정 (잔존율 기준·OV 로케이션 등)
create table app_settings (
  key        text primary key,
  value      jsonb not null,
  updated_at timestamptz default now()
);
```

4. 왼쪽 **Settings → API**:
   - **Project URL** 복사 → `SUPABASE_URL`
   - **anon / public** 키 복사 → `SUPABASE_KEY`

---

## STEP 2 — GitHub에 코드 올리기

이 폴더(`designated-shipment`)를 **private** 저장소로 push:

```bash
cd designated-shipment
git init
git add .
git commit -m "init: OV5 지정출고 (Supabase 공유)"
git remote add origin https://github.com/내ID/ov5-shipment.git
git push -u origin main
```

> `.gitignore`가 비밀키·dist·임시파일을 자동 제외합니다.
> `config/master_info.xlsx`(기준정보·하대)는 포함되어 기본값으로 쓰입니다.

---

## STEP 3 — Streamlit Cloud 배포

1. https://share.streamlit.io → GitHub 로그인 → **New app**
   - Repository: `내ID/ov5-shipment`
   - Branch: `main`
   - **Main file path: `app_streamlit.py`**
2. **Advanced settings → Secrets** 에 붙여넣기:

```toml
SUPABASE_URL = "https://xxxx.supabase.co"
SUPABASE_KEY = "eyJ..."
```

3. **Deploy** → 1~2분 후 URL 생성. 사이드바에 **🟢 공유 모드 · Supabase** 표시되면 성공.

---

## STEP 4 — 기존 로트 데이터 이관 (선택, 1회)

지금까지 쌓인 수동/자동 로트를 클라우드로 옮기려면, 내 PC에서 한 번만:

```bash
# PowerShell
$env:SUPABASE_URL="https://xxxx.supabase.co"
$env:SUPABASE_KEY="eyJ..."
python seed_supabase.py
```

→ `config/lot_assignments*.xlsx` + `settings.json` 내용이 Supabase로 들어갑니다.
(이관 안 해도, 앱에서 로트 xlsx를 다시 업로드하면 됩니다.)

---

## 사용 흐름 (직원들)

1. URL 접속 → 재고 파일 업로드 → 현황판
2. 행별 카테고리 체크(농협/대리점/FS/MS/급식/소재) — **여러 명이 봐도 같은 데이터**
3. 주문 파일 업로드 → 자동 매칭
4. 다운로드 클릭 → 화면 체크 + 매칭결과가 **Supabase에 누적 저장**
   → 다음에 누가 들어와도 그대로 체크되어 있음

## 데이터 관리
- 로트 지정: Supabase → Table Editor → `lot_assignments` 에서 직접 확인/수정
- 기준정보(하대): 사이드바에서 새 `Item_*.xlsx` 업로드 (재배포 전까지 유지).
  영구 반영하려면 새 master_info.xlsx 를 커밋하거나 사내 LAN 앱에서 갱신.

## 참고 — 로컬/LAN 으로도 그대로
비밀키를 안 넣으면 **로컬 파일 모드**라, 기존 `실행.bat`/데스크톱(app.py) 그대로 사용 가능합니다.
